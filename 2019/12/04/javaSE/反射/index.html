反射

什么是反射

官方介绍

JAVA反射机制是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意方法和属性；这种动态获取信息以及动态调用对象方法的功能称为java语言的反射机制。

通俗解释

把.class文件加载到内存，并创建对应的Class对象(这是第一步)，然后获取Class对象(字节码对象)中的Constructor,Field,Method（这是第二步）。并通过这三个对象任意操作内存中的对象属性和行为（这是第三步）。

名词解释：

Constructor 构造方法描述器

Field 属性描述器

Method 方法描述器

注解

概念

注解就是一个标记，该标记给编译器看的。并且需要配合反射使用才能生效

格式

注意事项

如果注解中的属性没有默认值，那么在使用该注解的时候必须给默认值

如果注解只有一个value属性，那么在使用该注解的时候给value属性赋值时可以省略value

如果给注解中的数组属性赋值，格式 属性名={,,}

元昊:描述元昊的数据

元数据：描述数据的数据

元注解：描述注解的注解

元注解

java中元注解有四个： @Retention @Target @Document @Inherited；

　　 1.@Retention：注解的保留位置　　　　　　　　　

　　　　　　@Retention(RetentionPolicy.SOURCE) //注解仅存在于源码中，在class字节码文件中不包含

　　　　　　@Retention(RetentionPolicy.CLASS) // 默认的保留策略，注解会在class字节码文件中存在，但运行时无法获得，

　　　　　　@Retention(RetentionPolicy.RUNTIME) // 注解会在class字节码文件中存在，在运行时可以通过反射获取到

　　

　　2.@Target:注解的作用目标

　　　　　　　　@Target(ElementType.TYPE) //接口、类、枚举、注解

　　　　　　　　@Target(ElementType.FIELD) //字段、枚举的常量

　　　　　　　　@Target(ElementType.METHOD) //方法

　　　　　　　　@Target(ElementType.PARAMETER) //方法参数

　　　　　　　　@Target(ElementType.CONSTRUCTOR) //构造函数

　　　　　　　　@Target(ElementType.LOCAL_VARIABLE)//局部变量

　　　　　　　　@Target(ElementType.ANNOTATION_TYPE)//注解

　　　　　　　　@Target(ElementType.PACKAGE) ///包

​ 3.@Document：说明该注解将被包含在javadoc中

　 4.@Inherited：说明子类可以继承父类中的该注解

总结

1.junit测试

​ 编写方法

​ @Test

​ public void 方法名() throws Exception{

​ }

2.反射

​ 通过另一个中方式操作对象的属性，方法，构造方法

​ 在程序运行过程中，动态的获取其构造方法，方法，属性，注解

​ 通用书写步骤(无论是私有的，还是非私有的)

​ 1.获取字节码对象

​ 2.获取构造方法/成员方法/属性,getDeclaredXXXX

​ 3.设置访问权限 setAccessible(true)

​ 4.xx不确定

3.注解(了解)

​ 两种使用方式

​ 1.作为配置文件使用

​ 2.作为标记

​ 定义格式

​ @Retention(RetentionPolicy.RUNTIME)//此句必须加

​ public @interface A{

​ 数据类型 属性名() default 属性值;

​ }

​ 使用

​ 可以在类，方法，属性，形参

​ @A

​ public class Test{

​ @A

​ private String name;

​ @A

​ public void show(@A String a){

​ }

​ }

​ 注解肯定配合反射使用

​ Class clazz = PersonTest.class;

​ Q q= (Q)clazz.getDeclaredAnnotation(Q.class);

​ q.age();//获取注解中的属性

​ 接下来要根据属性的值进行判断。。。
